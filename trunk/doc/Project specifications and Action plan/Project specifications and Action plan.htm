Introduction - Users and uses

The aim of our project is to create a web application bound to students and teachers, intended to create slideshows. A slideshows is a set of transparency (containing basically text, images and charts) presented one after the other to an audience for an instructionnal purposes.

Students and teachers will be able to log in the application, create a slideshow or edit existing ones, as well as using the software to play the slideshow when it is finished.

Why another presentation program ?

There is already, among desktop as well as web application, many existing solution to create presentations

Desktop full-featured applications

The most known of those softwares are the desktop ones, such as Microsoft PowerPoint and OpenOffice.org Impress. Their particularity is to have a larga amount of features (and, coming hands in hands, a large amount of toolbars, menus and sub-menus), allowing to customize each and every part of the slides : the transitions between slides, the animations, size, color, shape, inclination, ... of the smallest element of the slide.
They are appropriate to produce professional presentation, corresponding exactly to user's needs and desires. Until now they are the best tools for physical presentations, as they allows to have a presentor view of the slideshow on a screen (with comments and slides preview) and the normal view on another media, like a video projector.

On-line collaborative applications

More and more web-based office are offering the possibility to create presentations inside a web browser and to store it online. Their particularity is to make the presentation available from everywhere in the world, providing the user has an internet access and a web browser. It's no more required to have a (heavy) application installed localy, nor to take care of software updates.
Most of these application features a version control and allows many users to edit the document at the same time in a collaborative way and to present it online, but they are not designed to make an offline presentation : there is now way to have two different views on two different medias.
They are appropriate to create basic documents with a limited amount of animations/transitions and types of embedable elements (no video, music).

On-line sharing applications

There is another, less known web-based kind of tool in relation with our project : the presentation sharing platforms. They are used to manage and shares collection of slideshows on the web by producing a presentation applet out of an uploaded slideshow (a kind of Flickr).

Our approach

Our application can not compete with the number of features of desktop presentation program and our primary goal is not to produce a collaboration tool either.
We have to design an alternative to traditional tools: allowing any user from everywhere to easily create and play visually attractive presentations.

Fonctionnal specifications

must

Our interface will implement basic features like create/open/save a presentation. Each user must be identified by logging in thanks to a personal account. As he/she is successfully logged in, the list of his/her presentation set is shown through thumbnails.
When editing a show, the user has access to many tools in order to edit it. First, it begins with the creation of slides. Each added slide is removable by deleting it from the slides browser, which provide a global view of the whole presentation. As a slide is edited, many elements can be added such as texts, pictures, graphics or links. Each element can be modified and deleted. Copy, cut, paste functions will be available, as well. Between each slides, the user can add a slide transition.
Otherwise, the application will implement an undo/redo function in order to getting through the command history.
As creating a new presentation, a choice of themes is available. Each slide can follow a template selected first, to manage the page setting.
In addition, a timeline allows the user to add animation effects on its elements. These animations might be fade in/out, moving top/bottom, scale,… To manage elements, the depth level of each one can be modified.
Finally, a show is viewable in a full-screen mode.

should

The application proposes revision tools to get through the creation history of the presentation. In addition, a document is printable in different formats such as thumbnails, notes and slides, writing lines, using css stylesheets. The comments are proposed, as well.
A stand alone version of the application is suggested.

may

The application proposes collaboration tools in order to share presentations and to work on it in a group. A link to the presentation is sent by mail. Throughout the show, the computer screen shows a Presenter mode where the user can choose the panels he wants to see, like: time, duration, next slide, notes.
In the editor mode, the user can rotate each kind of elements. The open/save menu can be enhanced by features like import from/export to PowerPoint and PDF. A slide can be improved by importing files such as movies, sounds or flash. To perform fine slides, put elements in place is guided by fixed points (grid, rule) and the user can scale the view by zooming in/out. The whole interface is customisable by switching, resizing, moving the different widgets by dint of drag’s drop.
